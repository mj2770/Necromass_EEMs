Kmethod analysis 
	•	Overall view: Clustering algorithm for EEM datasets by leveraging PARAFAC and NMF-type models as base models inside each cluster
	•	Cluster EEMs samples so that each cluster have one PARAFAC model (instead of a global model for all samples) 
	•	Iteratively alternates
	•	Estimation: fit a base model first separately in each cluster
	•	Assignmnet: reassign each sample to the cluster whose model best explains it lowest distance/error
	•	Repeats base clustering with subsampling to build a consensus matrix indicating how often sample pairs co-cluster 
	•	Use hierarchical clustering on the consensus-derived distance matrix to produce a final clustering and final cluster-specific PARAFAC models 
	•	Predict method: assign any new EEMs data to the existing cluster-specific models

 KMethod.base_clustering
	•	Purpose: run the cluster once to obtain the cluster labels, per-iteration histories and cluster-specific models, guess an initial grouping, fit one model per group, test every sample against every group-model, move each sample to the group whose model fits it best, repeat until the clustering becomes stable
	•	Input:
	•	Eem_dataset: including eem_stack(3S array), sample index, splitting N#, subsampling (used indirectly via calculate consensus), ex and em range,
	•	Outputs:
	•	cluster_labels : final cluster assignment for each sample
	•	Label_histroy, error history
	•	Key steps and logic of the function
	•	Fit a global model to the whole dataset: baseline comparison, also to find the unified component, because after separately fit, component 1 in cluster A might actually be component 2 in cluster B, so global model gives common reference frame
	•	Initial guess of clustering: either random splits or based on quenching pairs 
	•	Fit one model per cluster: copy the base model and fit for each cluster
	•	Test every sample against every cluster-model and calculate the error, assign samples to the cluster that the fitted model gives the smallest error
	•	Regroup the samples 
	•	Remove clusters that are too small (removed undersized clusters)
	•	Align components across cluster models: align components 1 in cluster A to component 2 in cluster B
	•	Check the converged clustering
	•	Stop iterating if samples are assigned to the same clusters or if the models from each cluster is similar between each iterations
	•	Distance_metric to evaluate the fit quality 
	•	Reconstruction_error: reconstruct EEM, and this algorithm measures how different the reconstruction is from the actual sample 
	•	Reconstruction_error_with_beta: this is to handle the quenced/unquenched samples
	•	Quenching_coefficient: compares quenching-related coefficient to estimated from paired samples
	•	Other functions
	•	Get_quenching_coef: compare quenching and unquenching sample ratios
	•	Maximization: assign samples to best-fitting cluster
	•	Model_similarity: compare model across iterations using component similarity

KMethod.calculate_consensus
	•	Purpose: run clustering runs on subsamples and accumulate co-occurrence statistics and build a consensus matrix telling how often pairs of samples end up in the same cluster. Instead of asking what cluster is sample A, asking across repeated runs, how stable it is when sample A and sample B are in the same cluster
	•	Logic for doing this: cluster can be impacted by random initialization, subsampling, small data differences, so instead of just run one time, it reperates clustering many times to build a consensus matrix
	•	Input: eem_dataset, n_base_clustering, subsampling_portion (fraction of samples to retain in each subsample)
	•	Outputs: consensus_matrix, label history and error history
	•	This will be a value of sample pairs = 1: always cluster together, near 0: never clustered together
	•	Consensus = times clustered together / total observation time 
	•	Key steps
	•	Create two empty counting matrices: one for co-occurence matrix, count for times that sample pairs are split into a same subsample, one for label matrix counting how many times sample pairs are clustered together finally
	•	Repeat subsample + clustering many times: 
	•	Random subset
	•	Run base_clustering on that subset
	•	Update the pairwise counts
	•	Update pairwise counts
	•	Make sure coverage is enough: check whether some sample pairs never observed together
	•	Compute the consensus matrix

	•	The unified PARAFAC model refers to the PARAFAC model established on an entire EEM dataset without considering potential component shifts. Two unified PARAFAC models were established on P1 and P2 respectively (Table 2). PARAFAC was conducted by adopting the python package Tensorly (Kossaifi et al., 2019). The EEM dataset was represented by components (the vector outer product of excitation and emission loadings) and scores. The optimal number of components for the PARAFAC model was four according to variance explained and core-consistency (Bro and Kiers, 2003). The reconstruction error of each EEM was determined by calculating the reconstruction error.

After a unified PARAFAC model was established, the Fmax of each component was calculated by multiplying the component score with the maximum excitation and emission loadings (Murphy et al., 2011). 

Among the four components (C1, C2, C3 and C4), C1 is the only component that highly overlaps with the fluorescence fingerprint of bacteria, and its Fmax also exhibited the highest positive Pearson correlation coefficient with TCC.

Traditional steps for EEMs
	•	Read the standard output csv files
	•	Remove the background (just water samples) 
	•	Correst instrument response (raw intensity can be wave length biased
	•	Scatter removal /masking: Correct with Raman (inelastic scatter from water) and Rayleigh (elastic scatter)
	•	This will mask the scatter bands (set to NA) usually +-10-20nm around the band
	•	Inner filter effect correction 
	•	If the sample absorbs light strongly, fluorescence gets distorted, excitation is attenuated on the way in while emitted light is re-absorbed on the way out
	•	Standard approach is to measure UV-Vis and apply an IFE correction to calculate the F_corrected
	•	Normalize intensities (raman normalization) 
	•	Convert the intensity ro Raman units by normalizeing to the area or height of the water Raman peak
	•	This is to enabling data comparable across days/instruments/settings
	•	Find the peak and the region by set-up a number of components then trial and error to see the figures of lines of Excitation and emission 
	•	Traditional methods:
	•	Peak regions (fixed windows) picking (those Humic-like and protein-like regions defined by a Ex/Em windows
	•	Manual peak could inject error or overlap when multiple flurophores contribute 
	•	Calculate indices: such as FI fluorescence index, BIX, HIX
	•	Report the peak intensities (integrated region intensities)
	•	Parafac
	•	Prepare a 3D array with Samples X emission X excitation
	•	Choose candidate component numbers (2-8)
	•	Fit Parafac models and evaluate:
	•	residuals/explained variance
	•	Interpretability of component spectra
	•	stability/uniqueness
	•	Validation test
	•	Split-half analysis
	•	Split samples into halves, fit models check whether cmoponents match across splits
	•	Calculate the Fmax of each samples 
	•	Estimated maximum fluorescence of a component for each sample (regard as the amount)
